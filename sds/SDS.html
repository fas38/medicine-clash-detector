1. Architectural Pattern

For this project, we are using Model View View-Model (MVVM) pattern. It has three parts:

  • Model
  • View
  • View-Model





  • View: The user interface that is activities that are visible to the user and user input options are going to be included in View. View will observe for variables and actions exposed by View-models and provide the UI according to that.
  • View-Model: View will interact with database thorough view-model. All the logics and data-manipulation are going to be included in view-model.
  • Model: The model handles the Data of the application. It is not tied to the view or view-model, and because of this, it lessens the complexity.
  • Repository: We will have a repository class through which view-model will access model and server data.

We are using MVVM patterb in the project as it helps to write - manageable, maintainable, cleaner and testable code. MVVM is also supported by Google itself and many first-party libraries like lifecycle-aware components, LiveData, ViewModel and many more are available which we are going to use in our application.


We are using Room as object mapping library which is an ORM for SQLite db. It provides compile time checks of SQLite statements and can return LiveData observables.




2. Component Interactions

Our selected platforms are IOS and Android. For android app,we are going to use kotlin and for ios app, we are going to use swift. We are also using Google's "Mobile Vision API" to read text from images.

In database,we are using only local database. So online connection is not required to run both the IOS and Android version of the app. Online connection is only required while registering or logging in to the app; but to use the app online connection is not necessary.

3.2 Details of Class

  • UserInfo

    UserInfo class has name, username, password, email, image,user_id attributes.

  • ClashReport

    ClashReport class has medicine_1, medicine_2, clash_reason, user_id attributes.

  • DrugList

    DrugList class has drug_name,user_id attributes.

  • MedicalCondition

    MedicalCondition class has type, prohibited_drugs, user_id attributes.
    It has also four sub-class:

    • Heart Disease: This sub-class is for heart disease condition.
    • Kidney Disease: This sub-class is for kidney disease condition.
    • High Blood Pressure: This sub-class is for high blood pressure condition.
    • Diabetes: This sub-class is for diabetes related condition.

  • Medicine

    Medicine class has name, generic_name, brand_name, precaution, incompitable_drugs attributes.
    It has also four sub-class:

    • Antipyretics: This sub-class is for drugs related to reducing fever.
    • Analgesics: This sub-class is for drugs related to reducing pain.
    • Antibiotics: This sub-class is for drugs related to inhibiting germ growth.
    • Stimulants: This sub-class is for drugs related to stimulants.
